# Refactoring Catalog / 重構方法清單

這個檔案彙總了一些常見重構手法及適用場景。

## How to Use

1. 先識別 smell
2. 找對應 refactoring
3. 小步實施
4. 每一步都跑測試

## 常見 Refactorings

### Extract Method

**適合**：長函式、重複邏輯、需要給概念命名  
**做法**：

1. 抽出程式碼塊
2. 給新方法起清晰名字
3. 處理引數和返回值
4. 用新方法替換原始碼
5. 測試

### Inline Method

**適合**：方法體和名字一樣簡單，抽象沒有價值  
**做法**：

1. 找到所有呼叫點
2. 直接內聯
3. 刪除原方法
4. 測試

### Extract Variable

**適合**：複雜表示式難懂  
**做法**：

1. 抽出中間變數
2. 用有意義命名
3. 替換原表示式
4. 測試

### Rename Variable / Function

**適合**：命名錶達不清  
**做法**：

1. 明確真正職責
2. 批次替換
3. 測試
